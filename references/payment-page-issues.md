# 支付页面已知问题与修复记录

## 双重「支付成功」提示

**问题**：`#keyDisplay` 里写死了 `✅ 支付成功！`（第95行），同时轮询成功又在 `#statusText` 设了同样内容，两个同时显示。

**修复**：删掉 `#keyDisplay` 里写死的成功提示，只保留轮询动态设置的 `#statusText`。

## 终身畅享套餐显示「0天」

**问题**：价格展示逻辑（第74行）只处理了 `count` 和 `lifetime` 类型，`lifetime_daily` 掉到 else 分支，显示 `/ 0天`。

**修复**：补充 `lifetime_daily` 分支，显示 `/ 每日限150次`。

## 查询接口只支持本地订单号

**问题**：`query.php` 只按 `order_id`（本地订单号）搜索，用户输入支付宝交易号查不到。

**修复**：SQL 改为 `WHERE o.order_id = ? OR o.alipay_trade_no = ?`，输入订单号或支付宝交易号均可查询。

## 首页导航栏重复

**问题**：`index.html` 的 `</div>` 闭合全部错乱，`.footer` 被重复嵌入每个代码示例内部，「购买 API 密钥」链接出现十几次。

**修复**：完全重写 HTML 结构，每个 `.card` 独立闭合，购买按钮独立卡片展示。
